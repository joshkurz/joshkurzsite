## Critical Vulnerabilities

**Rate limiting is completely disabled**
`middleware.js:73-75`

```js
export const config = {
  matcher: [],  // matches NOTHING ‚Äî middleware never runs
}
```

The `LIMITS` map correctly defines 10 req/min for `/api/ai-joke`, 10 for `/api/speak`, and 5 for `/api/custom-jokes` ‚Äî but `matcher: []` means Next.js never invokes the middleware for any route. Every rate limit check is dead code.

**Attack scenario:** An attacker can call `/api/ai-joke` or `/api/speak` in a tight loop with no throttle, burning the entire OpenAI quota in seconds. `/api/custom-jokes` can be spammed to flood DynamoDB with rejected-joke records.

**Fix:**
```js
export const config = {
  matcher: ['/api/ai-joke', '/api/speak', '/api/custom-jokes'],
}
```

## Security Weaknesses

**1. `/api/author-jokes` ‚Äî no validation on `author` query param**
`pages/api/author-jokes.js:20-36`

```js
const { author } = req.query  // could be string[] if ?author=x&author=y
// ...
':pk': `AUTHOR#${author}`   // array.toString() yields "AUTHOR#x,y"
```

There is a null check but no length cap and no array guard. A client sending `?author=<4000-char string>` or `?author=x&author=y` gets unexpected behavior. Not injectable (ExpressionAttributeValues is parameterized), but long strings consume DynamoDB read capacity and array coercion yields a nonsense GSI lookup silently.

**Fix:**
```diff
  const { author } = req.query
- if (!author) {
+ if (!author || typeof author !== 'string' || author.length > 200) {
    return res.status(400).json({ error: 'Author parameter is required' })
  }
```

**2. `/api/ratings` GET is unrated ‚Äî even if middleware were fixed**
`middleware.js:5-9` ‚Äî `LIMITS` does not include `/api/ratings`.

Anyone can call `GET /api/ratings?jokeId=x` at unlimited rate. Each call is a DynamoDB GetItem so blast radius is limited, but it is an unprotected read endpoint.

**3. Missing Content-Security-Policy**
`next.config.js:5-16`

X-Frame-Options, X-Content-Type-Options, and Referrer-Policy are set globally, but there is no `Content-Security-Policy` header. If an XSS vector were ever introduced, there is no browser-side last line of defense.

**4. GHSA-fj3w-jwp8-x2g3 ‚Äî fast-xml-parser stack overflow (LOW)**

Installed: `fast-xml-parser@5.3.6` via `@aws-sdk/xml-builder@3.972.8`
Safe threshold: `>= 5.3.8`
`@aws-sdk/xml-builder@latest` already ships `5.4.1` ‚Äî fix is available upstream.

```bash
npm audit fix
```

## Security Wins

- No hardcoded secrets ‚Äî all credentials via `process.env`, `.gitignore` excludes `.env*`
- DynamoDB parameterized queries ‚Äî ExpressionAttributeValues used throughout; no string-interpolated expressions
- Field validation ‚Äî `custom-jokes` and `ratings` both enforce type checks, length limits, and integer range checks on all user inputs
- AI content moderation ‚Äî user-submitted jokes are passed through GPT moderation before storage
- Voice allowlist ‚Äî `speak.js:6-7` validates `voice` against an explicit allowlist, no passthrough
- Security headers ‚Äî X-Frame-Options: DENY, X-Content-Type-Options: nosniff, Referrer-Policy globally applied
- No `dangerouslySetInnerHTML` ‚Äî React JSX escaping is the only rendering path
